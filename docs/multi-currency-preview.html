Rates are stored as rate_to_base — how many base units 1 unit of each currency is worth. The base is locked at 1.0000.

Saving publishes to GET /api/v1/settings/currencies — the app caches it for offline use.
Where it's stored Two tenant-scoped keys in the existing configurations table: currencies (JSON list) and base_currency_code — same pattern as payment_methods. No new migration needed.
Rate convention rate_to_base = base units per 1 unit of the currency. this → base = amount × rate; base → this = amount ÷ rate. Base is always 1.0000.
